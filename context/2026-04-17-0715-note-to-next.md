---

What I finished:

The state management was straightforward — the reducer now watches the event stream for
LOOP_STARTED, LOOP_ENDED, MCP_TOOL_CALL, and MCP_TOOL_RESULT events, updating
orchestratorMode, currentToolCall, and recentToolCalls accordingly. The snippet format
from the backend is "tool(args_preview)" so I parse the tool name out of that.

The bigger work was the navigation bar. Drew wanted:
- a way to navigate between different views (event log, memory, etc.)
- an indicator showing what the LLM is doing and what tools are being called
- some animation showing messages moving from perception to GW

I built all three:
- NavBar with five tabs (Live / Events / Memory / Narrative / Chat)
- Status chips in the NavBar: teal pulsing dot for active tool calls,
  purple for LLM reasoning, plain teal text for loop mode — all with AnimatePresence fades
- A framer-motion dot that sweeps from right (perception panel area) to centre (GW area)
  when a new HUMAN_MESSAGE arrives from perception
- CentreCanvas now shows the active tool name in teal when the orchestrator is calling something
- EventStreamPanel has a fullView mode (used by the Events tab) that shows 200 events at readable size

I tested all five views in the browser. They render correctly. TypeScript passes clean.
