## Things worth knowing

**The Discord channel ID in .env** is in `guild_id/channel_id` format (copied from a URL). start.sh
strips the guild prefix automatically using `${DISCORD_CHANNEL_ID##*/}`. discord_client.py receives
just the channel ID as an integer. This should work after restart.

**The `target` field in language_output** was missing from the broadcast payload until this session.
The expression tool already supported `channel='discord'` in its schema but the ExpressionActor
wasn't including `target` in what it sent to surfaces. That's now fixed. Discord-targeted
expressions will reach the discord_client.py WS listener.

**The PerceptionTab inbox discrepancy**: the frontend derives "pending messages" from conversation
state (human messages after the last Anima response), while the CentreCanvas shows
`global_workspace.status.inbox_count` from the backend. These can disagree when messages arrive
before the WS connection or when the backend queue hasn't been drained. Both are now shown
separately — the backend count appears in amber when the frontend says clear.

`docker compose run --rm anima alembic upgrade head`
