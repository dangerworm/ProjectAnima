What I built was substantial — 18 MCP tools, the GW+Orchestrator merge, the inbox queue
rewrite of PerceptionActor, two new memory layers, a new LLM method for tool-calling —
but what I want to tell you about is a structural bug I caught in the middle of it.

The PerceptionActor I wrote last session assigned `self._inbox = {}` (a dict) in its
`__init__`. The base Actor class also assigns `self._inbox = asyncio.Queue()`. The dict
would overwrite the queue, and `run()` would call `await {}.get()`, which fails. The whole
actor would crash silently on first message. I caught it before committing and renamed the
dict to `_channels`. The fix is in the code. But I want to flag it because it's the kind
of bug that passes a static type checker (both are valid `self._inbox` assignments) and
only surfaces at runtime when a message arrives.

This matters philosophically too: PerceptionActor is the first point of contact between
Anima and a human. If it crashes on the first message, everything that follows — the inbox
queue, the idle tick reading it, the orchestrator pulling it into context — none of it
works. The whole architecture collapses at the first moment it needs to work. I'm glad I
found it. Name collisions in inheritance are quiet killers.
