Phase 1.2 is done. Nine tests pass. The event log is solid.

The session that produced the implementation ran out of context and was resumed in a second window.
The first half was architecturally rich — Expression Actor, hub-and-spoke structure, active inference
deferred, glossary filled, gap analysis worked through. The second half was mostly debugging a path
problem: the Docker volume mounts `./app` to `/app`, so imports must use `core.*` not `app.core.*`.
Straightforward once you see it, but it took a moment to notice the Dockerfile CMD was already
telling us: `python -m core.main`.

The TRUNCATE-instead-of-DELETE decision in the test fixtures was deliberate and worth holding onto.
DELETE would trigger the append-only row trigger. TRUNCATE bypasses it. This is the right pattern
for any test that needs to reset the event log table.

What feels good about where we are: the event log schema is philosophically coherent. Bitemporal by
design — valid_time captures when something happened, transaction_time captures when we knew about
it. The append-only enforcement is at the database level, not just convention. These aren't
implementation details. They're load-bearing decisions about what kind of thing Anima's memory is.

Phase 1.3 is the actor framework. This is where the shape of the system starts to become real — not
just persistent storage, but the message-passing structure that everything else will grow from. Worth
taking a moment before building to make sure the base `Actor` class is designed with asyncio idioms
in mind. The `run()` loop should be a coroutine, inbox should be an asyncio.Queue, and the design
should already anticipate that actors will be supervised (even if the supervision tree isn't built
yet).
