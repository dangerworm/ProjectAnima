# Web UI Ideas

## Global Workspace (top centre)

Possible icon: 🧠

This is the thalamus of the system. We should see all the incoming events.

Upgrade ideas:

- Animated signal cards entering from edges
- Cards pulse based on salience
- When ignition happens:
  - Screen briefly "flashes"
  - Winning signal expands and broadcasts outward
- Queue depth
- Ignition threshold line
- "losing" signals fade/die or move elsewhere

## Temporal Core (top left)

Possible icon: ⏱

Enhancements:

- Pulsing "heartbeat" dot
- Tick waveform (like a metronome trace)
  - "gap detection" flash (missed heartbeat = glitch flicker)
- Subtle drift animation when dormant

## Perception (top right - good placement)

Possible icon: 👁

Show all inputs modes as a short line of icons (see next list) as tabs that can be clicked

Make it multimodal:

- Switch tabs dynamically as new inputs come in:
  - 🎤 Audio → waveform + live transcript (if possible) scroll
  - 📷 Vision → last frame thumbnail
  - 🖥 X11 → mini screen capture
  - 🗨️ Text → incoming message cards

Animation ideas:

- "Scan lines" across images
- Audio waveform reacts in real time
- After a few seconds, inputs are minimised to an icon (a small box?) and move to global workspace

## Memory (left column)

Possible icon: 🧠

Break into sub-layers visually as small frames:

- Event Log: scrolling ticker showing last 3
- Identity: stable core block
- Reflective Memory: floating nodes / graph
- Volitional: decision ledger
- Residue: glitchy, distorted nodes

Animations:

- WRITE: glowing pulse → node appears
- READ: frame lights up
- Residue: jitter / static / never fully resolves
